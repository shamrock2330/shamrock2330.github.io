Git

  • Git是目前世界上最先进的分布式版本控制系统

Git的相关操作

a、使用前进行的全局配置

配置用户名

  • git config –global user.name “用户名”

配置邮箱

  • git config –global user.email “邮箱地址”

查看配置（最后两行就是用户名和邮箱）

  • git config -l

具体查看配置操作

  1. 查看系统的config
  • git config –system –list
  1. 查看当前用户（global）配置
  • git config –global –list

b、创建仓库：当我们需要让Git去管理某个新项目/已经存在的项目的时候，就需要创建仓库，最好是在一个空目录下去创建一个新的仓库。【最好不要使用包含中文的目录名】

  • mkdir 文件名【创建一个空目录】

  • cd 文件名【进入目录】

  • git init【Git仓库初始化：初始化之后会出现一个.git的文件夹】

c、Git常用指令操作

  • 查看当前状态：git status

  • 从远程仓库克隆 git clone <仓库路径> [本地目录]

  • 添加到缓存区：git add 文件名

    • 说明：git add指令可以添加一个文件，也可以同时添加多个文件

    • 1、git add 文件名

    • 2、git add 文件名1 文件名2…

    • 3、git add . 【添加当前目录到缓存区中】

  • 提交至版本库：git commit -m “注释内容”

d、版本回退

  • 1、查看版本，确定需要回到的时刻点，指令：git log / git log –pretty=online

  • 2、回退操作，指令：git reset –hard 版本号（提交编号）

  • 注意：回到过去之后，要想再回到之前最新版本的时候，则需要使用指令去查看历史操作，以得到最新的commit id 指令：git reflog

e、生成公钥指令：**ssh-keygen -t rsa -C “注册邮箱”**（该指令生成带有身份识别的密钥，管理多密钥时有用）

Git理论

Git本地有三个工作区域:工作目录（Working Directory）,暂存区（Stage/Index），资源库（Repository或GDirectory）。如果再加上远程的git仓库（Remove Directory）就可以分为四个工作区域。文件在这四个区域之间的转换关系如下：

  • Workspace:工作区，就是平时存放项目代码的地方
  • Index/Stage:暂存区，用于临时存放你的改动，事实上它是一个文件，保存即将提交的列表信息
  • local Repository:仓库区（或本地仓库），就是安全存放数据的位置，这里有你提交到所有版本的数据，其中HEAD指向最新放入仓库的版本
  • Remote Repository:远程仓库，托管代码的服务器，可以简单的认为是你项目组中的一台电脑用于远程数据交换

Git工作流程

git的工作流程一般是这样的：

  1. 在工作中添加，修改文件

  2. 将需要进行的版本管理的文件放入暂存区域

  3. 将暂存区域文件提交到git仓库

配置SSH公钥

1、生成SSH公钥（如果公钥已经存在，则自动覆盖）

  • ssh-keygen -t rsa

  • 不断回车（即使用默认设置）

2、Gitee设置账户公共钥（或者GitHub仓库的公钥）

  • 查看生成的公钥

    • cat ~/.ssh/id_rsa.pub

  • 验证是否配置成功

    • ssh -T 地址

    • 例如验证是否连接了GitHub：ssh -T git@github.com

    • 一定要注意使用该指令测试GitHub的连接时不要开Watt Toolkit，否则可能导致连接失败

3、设置ssh免密登录：

首先按照上述方法生成查看公钥并复制（需要进入对应的目录才能查看）

以下连续按三次回车（这是执行了ssh-keygen -t rsa生成公钥）

（也可以直接去到 C:\用户{你的用户名}.ssh 目录下查看id_rsa.pub并复制公钥）

接着在GitHub上面进行设置

弄完以后当前电脑客户端接下来再往远程仓库 push pull 东西的时候使用 SSH 连接就不需要登录了。

其他操作：

  • 查看与GitHub的连接操作：ssh -T git@github.com

  • 查看GitHub详细日志：ssh -vT git@github.com
